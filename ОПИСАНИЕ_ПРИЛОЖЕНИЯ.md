## 1. О приложении

**PWA «Финансовый учёт»** — веб-приложение для личного финансового учёта: доходы и расходы, счета, категории, переводы, планировщик операций, контакты, долги, отчёты и резервное копирование.

- Работает в браузере и устанавливается как PWA (Progressive Web App) на телефон или компьютер.
- Каждый пользователь видит только свои данные (изоляция через Supabase Auth + Row Level Security).
- Интерфейс на русском языке, адаптивный (десктоп + мобильная навигация).
- Поддерживается светлая и тёмная тема.

**Деплой:**
- Фронтенд — [Vercel](https://vercel.com) (каталог `frontend/`).
- Бэкенд и база данных — [Supabase](https://supabase.com) (PostgreSQL + Auth + RPC).

## 4. Разделы приложения (функционал)

### Аутентификация

| Маршрут | Описание |
|---------|----------|
| `/login` | Вход по email и паролю |
| `/register` | Регистрация |
| `/forgot-password` | Запрос ссылки для сброса пароля |
| `/reset-password` | Установка нового пароля по ссылке из письма |

После входа доступны все разделы под общим layout (боковое меню / нижняя панель на мобильном).

### Основные разделы

| Маршрут | Раздел | Возможности |
|---------|--------|-------------|
| `/dashboard` | **Дашборд** | Сводка доходов/расходов, баланс, графики (расходы по категориям, динамика баланса), топ транзакций, сравнение с прошлым периодом, **блок «Скоро к оплате»** (автопланировщик), быстрое добавление дохода/расхода |
| `/accounts` | **Счета** | Наличные, карты, накопления; валюта, иконка, начальный баланс, архивация |
| `/categories` | **Категории** | Дерево категорий доходов и расходов, сортировка, системные категории по умолчанию |
| `/transactions` | **Транзакции** | Список, создание/редактирование дохода и расхода, теги, примечание, фильтры (период, счёт, категория, тип, поиск), повтор транзакции |
| `/transfers` | **Переводы** | Перевод между счетами, комиссия, примечание |
| `/scheduled` | **Планировщик** | Регулярные операции (день, неделя, месяц и т.д.) |
| `/contacts` | **Контакты** | Имя, телефон, email, избранное; поиск по Enter / кнопке «Найти» |
| `/contacts/:id` | **История контакта** | Все долги и платежи с человеком, итог «мне должны / я должен», детали долга |
| `/debts` | **Долги** | «Я должен» / «Мне должны»; погашение и увеличение; редактирование операций; выбор счёта для транзакции |
| `/reports` | **Отчёты** | Сводка за период, разбивка по категориям, сравнение периодов, прогноз, топ операций; бэкап/восстановление JSON |
| `/settings` | **Настройки** | Имя, валюта по умолчанию, смена пароля, тема оформления, **автобэкап**, удаление аккаунта |

### Мобильный интерфейс

- Нижняя навигация: дашборд, транзакции, добавить, отчёты, «ещё».
- Лист «Ещё» — остальные разделы.
- FAB «+» на дашборде для быстрого добавления операции.
- Форма транзакций — sheet-модалка, одна колонка.

### Отчёты и данные

- **Бэкап** — выгрузка JSON (счета, категории, контакты, транзакции, переводы, долги и т.д.).
- **Восстановление** — загрузка JSON через RPC `restore_user_backup` (с лимитом размера файла ~15 МБ).
- **Экспорт Excel / PDF** — кнопки есть, реализация пока заглушка (сообщение пользователю).

## 5. Модель данных (таблицы Supabase)

| Таблица | Содержимое |
|---------|------------|

## 7. Локальный запуск и деплой

### Локально

```bash
cd frontend
cp .env.example .env
# Заполнить VITE_SUPABASE_URL и VITE_SUPABASE_ANON_KEY
npm install
npm run dev
```

Приложение: http://localhost:5173

### Сборка

```bash
cd frontend
npm run build
```

### Деплой на Vercel

1. Репозиторий подключён к Vercel, корень сборки — `frontend/`.
2. В настройках Vercel задать переменные `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.
3. `git push` — Vercel собирает и публикует автоматически.

SQL-миграции в Supabase применяются **отдельно** от деплоя фронтенда.

## 8. Улучшения, сделанные в ходе разработки

Ниже — изменения, которые вносились по ходу работы над проектом (не начальный MVP, а доработки).

### 8.1. Архитектура и миграция на Supabase

- Переход с Node.js/Express + Prisma на **Supabase** (PostgreSQL, Auth, RLS, RPC).
- Единый API-слой во фронтенде: `frontend/src/api/supabase.ts`.
- Маппинг snake_case ↔ camelCase в `supabaseMappers.ts`.

### 8.2. Безопасность

- **RLS** по внешним ключам и владельцу данных (`20250110_security_rls_and_rpc.sql`).
- Защита RPC для категорий и `restore_user_backup` от подмены `user_id`.
- Утилита **`postgrestFilter.ts`** — безопасные ILIKE- и UUID-фильтры (защита от filter injection).
- Заголовки **CSP, HSTS, X-Frame-Options** и др. в `vercel.json`.
- Лимиты на бэкап: размер файла (~15 МБ), число строк на таблицу.
- Секреты вынесены из репозитория: ключи только в `frontend/.env`, добавлен корневой `.gitignore`.

### 8.3. Настройки пользователя

- Страница **`/settings`**:
  - редактирование имени;
  - валюта по умолчанию (KZT, RUB, USD, EUR и др.);
  - смена пароля;
  - удаление аккаунта (с подтверждением email).
- SQL: `20250111_user_settings.sql` — колонка `default_currency`, RPC `delete_own_account()`.
- Профиль и валюта доступны через `AuthContext`.

### 8.4. Дашборд

- **Сравнение периодов** — доходы/расходы относительно прошлого месяца, недели или вчера (`PeriodComparisonCard`).
- **Быстрое добавление** на десктопе (кнопки доход/расход).
- **FAB «+»** на мобильном для быстрого создания операции (`QuickAddFab`).
- **Автопланировщик «Скоро к оплате»** (`UpcomingScheduledBlock`):
  - активные `scheduled_transactions` с датой выполнения в ближайшие 7 дней (включая просроченные);
  - кнопка **«Выполнить сейчас»** — RPC `execute_scheduled_now`, создаёт транзакцию и переносит следующую дату;
  - ссылка на полный раздел «Планировщик».
- Выбор периода: сегодня / неделя / месяц.

### 8.5. Транзакции

- **Повтор транзакции** — кнопка ↻ в списке, форма заполняется из выбранной операции (`repeatSource`).
- **Мобильная форма** — sheet-модалка, одна колонка полей.
- **Фильтры в URL** — период, счёт, категория, тип, поиск сохраняются в query-параметрах.
- **Поиск** — по примечанию и тегам (исправлен: раньше параметр `search` не обрабатывался в API).
- Поиск по **Enter** или кнопке **«Найти»** (не на каждый символ).

### 8.6. Контакты

- Поиск по имени, телефону, email через PostgREST ILIKE.
- Поиск по **Enter** / кнопке **«Найти»** (компонент `SearchField`).
- Фильтр «Избранные».

### 8.7. Тема оформления

- **Светлая / тёмная / как в системе** (`ThemeContext`, `utils/theme.ts`).
- Переключатель в настройках.
- Поддержка `prefers-color-scheme`.
- Стили sidebar, header, карточек для dark mode.

### 8.8. Восстановление пароля

- Страницы `/forgot-password` и `/reset-password`.
- Ссылка «Забыли пароль?» на странице входа.
- Redirect URL `/reset-password` должен быть в Supabase Dashboard.

### 8.9. Исправления сборки и UI

- Замена несуществующего цвета `primary-950` на `primary-900` в Tailwind.
- Исправлен циклический `@apply` в `globals.css` (использование `theme()` / `rgb()`).
- Общий компонент **`SearchField`** для единообразного поиска.

### 8.12. Автобэкап по расписанию

- **Еженедельное напоминание** (7 дней): баннер «Сделайте бэкап» вверху приложения (`BackupReminderBanner`).
- Кнопки: создать бэкап, перейти в отчёты, отложить на 3 дня.
- **Автосохранение** (опционально в настройках): при входе в приложение, если бэкап просрочен — автоматическое создание файла (один раз за сессию).
- **Экспорт файла** (`backupExport.ts`):
  - на мобильном — **Web Share API** (сохранить в «Файлы», Telegram, Drive и т.д.);
  - на десктопе — скачивание `.json`.
- Настройки в **`/settings`** → блок «Автобэкап»: напоминание, автосохранение, дата последнего бэкапа.
- Утилиты: `backupSchedule.ts` (localStorage), хук `useBackupScheduler`.

### 8.11. История по контакту

- Страница **`/contacts/:contactId`** (`ContactDetailPage`):
  - карточка контакта (телефон, email, заметка);
  - **итог по открытым долгам** — «Мне должны», «Я должен», чистая позиция (по каждой валюте отдельно);
  - список **всех долгов** (все статусы), клик — детали в модалке;
  - единая лента **всех операций** (погашения и увеличения) по всем долгам, отсортированная по дате;
  - из модалки долга — погашение, увеличение, редактирование и удаление операций.
- API: `contacts.getHistory(id)` + утилита `buildContactHistory` в `utils/contactHistory.ts`.
- Переход с карточки контакта в списке (клик по имени / «История →»).

### 8.10. Прочее (из ранних миграций)

- Надёжная регистрация пользователя и триггеры `user_id`.
- Категории по умолчанию при регистрации, сортировка и перестановка.
- Исправления неоднозначности в SQL-функциях долгов.
- Восстановление из бэкапа через RPC с fallback на клиентскую вставку.
