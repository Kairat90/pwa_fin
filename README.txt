ЧТО ЭТО
-------
Веб-приложение для личных финансов: доходы, расходы, счета, категории,
переводы, планировщик, контакты, долги, отчёты, бэкап.
Устанавливается как PWA. Интерфейс на русском. Светлая и тёмная тема.

СТЕК
----
Фронт:  React, TypeScript, Vite, Tailwind, TanStack Query, Supabase JS,
        Recharts, React Hook Form, Zod, vite-plugin-pwa
Бэкенд: Supabase (PostgreSQL, Auth, RLS, SQL-функции)
Деплой: Vercel (frontend/) + Supabase (база и auth)

СТРУКТУРА ПРОЕКТА
-----------------
pwa_fin/
  frontend/          — React-приложение
    src/api/         — API supabase.ts
    src/pages/       — страницы
    src/components/  — UI-компоненты
    src/context/     — Auth, Theme
    src/utils/       — валюта, даты, бэкап, фильтры
  supabase/
    migrations/      — SQL-миграции (применять в Supabase по порядку)

РАЗДЕЛЫ ПРИЛОЖЕНИЯ
------------------

/forgot-password            — запрос сброса пароля

/dashboard                  — сводка, графики, автопланировщик «Скоро к оплате», быстрое добавление
/accounts                   — счета
/categories                 — категории (дерево)
/transactions               — операции, фильтры, поиск, повтор
/transfers                  — переводы между счетами
/scheduled                  — регулярные операции
/contacts                   — контакты; /contacts/:id — история долгов и платежей
/debts                      — долги и платежи
/reports                    — отчёты, бэкап/восстановление
/settings                   — профиль, валюта, пароль, тема, автобэкап, удаление аккаунта

ЗАПУСК ЛОКАЛЬНО
---------------
cd frontend
cp .env.example .env
(заполнить VITE_SUPABASE_URL и VITE_SUPABASE_ANON_KEY)
npm install
npm run dev

УЛУЧШЕНИЯ (СДЕЛАННЫЕ В РАБОТЕ НАД ПРОЕКТОМ)
-------------------------------------------

1. Миграция на Supabase вместо Node.js backend

2. Безопасность:
   - RLS по владельцу данных
   - защита RPC и фильтров PostgREST (postgrestFilter.ts)
   - CSP, HSTS в vercel.json
   - лимиты на размер бэкапа
   - ключи только в frontend/.env

3. Страница настроек (/settings):
   - имя, валюта по умолчанию
   - смена пароля
   - удаление аккаунта
   - переключатель темы

4. Дашборд:
   - сравнение с прошлым периодом
   - FAB «+» на мобильном
   - быстрое добавление на десктопе

5. Транзакции:
   - повтор операции (кнопка ↻)
   - мобильная форма (sheet)
   - фильтры в URL
   - исправлен поиск по примечанию и тегам
   - поиск по Enter / кнопке «Найти»

6. Контакты:
   - поиск по Enter / кнопке «Найти» (не на каждый символ)

7. Тёмная тема (светлая / тёмная / как в системе)

8. Восстановление пароля (forgot-password, reset-password)

9. Исправления сборки (Tailwind, globals.css)

10. Автопланировщик на дашборде:
   - блок «Скоро к оплате» — активные операции на 7 дней
   - кнопка «Выполнить сейчас» (RPC execute_scheduled_now)
   - просроченные операции подсвечиваются

11. История по контакту (/contacts/:id):
   - все долги и платежи с человеком
   - итог «мне должны / я должен» по валютам
   - переход с карточки контакта

12. Автобэкап по расписанию:
   - напоминание раз в 7 дней (баннер «Сделайте бэкап»)
   - автосохранение файла при входе (опционально)
   - на мобильном — Web Share API (Поделиться)
   - настройки в /settings
